Fixes the "Failed to load runs" dashboard crash and adds a per-container plain (tar + xml) export, plus a clearer status for containers with no data folders. Thanks to **manilx** on the Unraid forum for the detailed testing and reports.

## Fixed
- **Run history could fail to load ("Failed to load runs").** A backup that was still running, or that was interrupted (BombVault restarted or crashed mid-backup), left a run with no byte count. That single record made the whole run-history query fail, so the dashboard's Run History showed "Failed to load runs" and never recovered. The history now reads such records correctly, so it loads again — no database reset needed.
- **Interrupted backups no longer linger as "running".** On startup, any run still marked "running" from a previous lifetime (only possible after a crash or update mid-backup) is now marked failed with an "interrupted" note, instead of showing a permanent "running" status on the dashboard.

## Added
- **Plain export (tar + xml), per container.** Each installed container now has an **Export** button next to Back up now. It writes a browsable, tool-free copy into an `export` folder next to your backup repository: `<name>.tar.gz` of the same folders restic backs up, plus the Unraid `<name>.xml` template — the same simple "one tar + one xml per container" layout as the Appdata Backup/Restore plugin. restic stays the backup engine; this export is an extra, **unencrypted** convenience copy you can open without any tool. (Containers only; VMs stay restic-only.)

## Changed
- **Clearer status for containers with no data folders.** A stateless container (nothing under appdata) no longer reports a plain "Done" that looks like a data backup. It now says **"Config only — no data folders"**: its definition/template is still captured so it can be recreated on restore, but it is clear that no data snapshot was made. Use the new Export button if you want the config/template as a file.

## Notes
- New strings (Export button, "Exported to", "Config only") are translated into all 26 UI languages.
- Export hardening: archive entry names can never escape the target tree, a failed export never leaves a truncated or misleading archive (atomic write), and overlapping selected folders are de-duplicated.
