Rounds out the backup toolkit with repository integrity checks and per-container pre/post-backup hooks, and gives credit where it's due.

## Added

### Integrity checks
- **Settings → Integrity**: run `restic check` per domain (Containers / VMs / Flash) to verify a repository's structure and metadata are intact. Friendly "no backups yet" when a repo hasn't been created; bounded by a timeout.

### Pre/post-backup hooks (per container)
- Each container can run a **pre-backup** and **post-backup** shell command (Containers → a container's *Backup hooks*). Commands run **inside the container** (`sh -c`).
- **Pre** runs while the container is still up — e.g. `mysqldump`/`pg_dump` into appdata so a consistent dump is included in the backup. A **failing pre-hook aborts the backup** (no inconsistent snapshot).
- **Post** runs after the container is back up (e.g. notify a webhook); its failure is logged but never fails the backup.
- Hooks are command execution by design; they run only inside the named, validated container and are gated by the same auth as every other action — no new capability over the Docker socket BombVault already uses.

BombVault is an independent Go + restic rewrite that extends the concept to VMs and the Unraid flash. See the README Credits section.

## Hardening
- Container exec (hook) output is read through a capped reader; JSON request bodies are size-limited (1 MiB) — both from the pre-release security audit (no blockers found).
