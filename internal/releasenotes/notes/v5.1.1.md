Fixes the "What's new" dialog on `:latest` builds.

## Fixed

- **The "What's new" dialog now finds its release notes on `:latest` builds.** `:latest` images carry a build version with extra metadata (e.g. `v5.1.0+main.<commit>`), and the dialog was using that whole string to look up the GitHub release — so it 404'd and offered a dead link. It now uses just the release tag (`v5.1.0`). As a bonus, the dialog no longer re-appears on every `:latest` rebuild: it pops once per actual release again, not once per commit. (#48)

Thanks to @manilx for the report.
