A big milestone release. Since BombVault went feature-complete on the 1.x line it gained a whole wave of UX, reliability and quality-of-life improvements — most of them driven by detailed Unraid forum testing. This 2.0.0 bundles all of it into a single release.

**Huge thanks to @manilx on the Unraid forums** for the relentless testing, the clear bug reports and the sharp feature ideas — a big part of this release exists because of that feedback. 🙏

## Added
- **Per-container backup-folder selection** — choose exactly which mapped folders to back up, plus custom paths under the host mount.
- **Backup notifications** — Webhook (Generic JSON, Discord, Slack, Gotify, ntfy), Matrix, or Healthchecks.io. Policy: never / on failure / always.
- **Stop other containers during a backup** — name dependent containers (e.g. a database) to stop while this one is backed up and start again afterwards, over the Docker API (no scripts). They always restart, even if the backup fails.
- **Plain export (tar + xml), per container** — an Export button writes a browsable, tool-free copy next to your repo: `<name>.tar.gz` of the backup folders plus the Unraid `<name>.xml` template (same idea as the Appdata Backup plugin). restic stays the engine; the export is an extra, unencrypted convenience copy.
- **Delete individual backups** — a Delete button on each snapshot (containers, VMs, flash) removes just that backup.
- **Repository maintenance** (Settings → Integrity & maintenance), per domain: **Verify** (`restic check`), **Unlock** (clear a stale lock left by an interrupted run — the “repository is already locked” fix), and **Prune** (reclaim space from deleted backups).
- **Collapsible folder tree** for file-level restore — expand/collapse like a file manager instead of one long flat list.
- **26-language UI.**

## Changed
- **VM live backups are reliable and never silently shut a VM down.** A live backup that hits a leftover BombVault snapshot overlay from a previously interrupted run now auto-commits that overlay and continues live; if a live snapshot genuinely can't be taken it fails with a clear message and leaves the VM running. (Use the explicit “graceful” method if you *want* a shutdown backup.) The per-VM method tooltip now spells out graceful = shutdown vs live = no downtime.
- **Backups never auto-create a share.** BombVault no longer creates `/mnt/user/bombvault` at first run (which Unraid turned into an undeletable share). The backup folder is created only when the first backup actually runs.
- **Live progress bar** across the full width at the bottom of each card.
- **Clearer status for data-less containers** — a container with no data folders reports “Config only” instead of a plain “Done”.
- New **white-framed logo** in the sidebar and browser tab.

## Fixed
- **Run history could fail to load (“Failed to load runs”)** — an interrupted run left a record the history couldn’t read; it loads again (no database reset, it was never corruption), and interrupted runs are tidied up on startup.
- **“Failed to load backups”** now self-heals a stale repository lock (clears it and retries).
- **Containers with no appdata no longer error** — they’re backed up as a definition-only backup (recreated on restore).
- **VM live snapshots skip cdrom / read-only disks** instead of failing on them.
- **The dashboard shows which container/VM each run was for, and the error on failure.**
- Same-repo operations are serialised, so a backup, prune or unlock can never run over each other.
