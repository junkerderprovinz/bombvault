Bug-fix release from the first round of Community-Apps feedback — thanks!

## Fixed
- **VM backup failed with a 400 for VMs whose name contains a space** (e.g. "Windows 11", "Home Assistant"). The name check was the strict Docker-container rule, which rejects spaces; VM names now use a libvirt-aware validator (spaces allowed; traversal / option-injection still blocked). Backup/restore of spaced-name VMs works now.
- **NVRAM transfer over SSH for spaced VM names.** The UEFI NVRAM path is derived from the VM name (…/`Windows 11_VARS.fd`); the remote command args are now shell-quoted so the path survives the remote shell instead of splitting — so a spaced-name UEFI VM keeps its exact NVRAM on restore.

## Changed
- **Transparency for the backup location.** The CA description and README now state that backups default to `/mnt/user/bombvault/` (containers/vms/flash), created on the first backup and changeable any time under Settings → Backup paths — so nothing appears on your array unannounced.

Image: `ghcr.io/junkerderprovinz/bombvault:1.3.4` (also `:1`, `:1.3`, `:latest`), multi-arch (amd64 + arm64).
