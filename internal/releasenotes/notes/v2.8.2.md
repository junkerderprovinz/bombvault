Two small dashboard/VM fixes from @manilx's feedback. Thanks.

## Added
- **Remove a stale VM entry.** A VM that was deleted from the host but still shows under "Not installed" with no backups now has a **Remove entry** button (VMs tab) that clears the leftover entry. (A VM that still has backups is handled by "Delete all backups" in its Backups panel, as before.)

## Fixed
- **Maintenance "healthy" indicator no longer carries across repos.** In Settings → Integrity & maintenance, switching the Local/Off-site source now clears the per-action result indicators, so a green result from one repo doesn't stay shown for the other where nothing has run.

> Note: the recurring daily `dumpxml: failed to get domain` error for a deleted VM was already fixed in v2.8.x (the scheduler now skips a VM that's no longer defined on the host) — updating to this image and removing the stale entry clears it for good.

Image: `ghcr.io/junkerderprovinz/bombvault:2.8.2` (also `:2`, `:2.8`, `:latest`) and `docker.io/junkerderprovinz/bombvault:2.8.2`, multi-arch (amd64 + arm64).
