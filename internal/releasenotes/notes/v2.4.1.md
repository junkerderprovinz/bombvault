Bug fixes for the 2.4.0 off-site features, from **@manilx**'s testing. Thank you.

## Fixed
- **Off-site view showed nothing for remote repos.** Switching a backup browser (or Integrity & Maintenance) to **Off-site** listed nothing when the off-site repo was a remote backend (`rest:`/`s3:`/`b2:`/`sftp:`), even though the data was there. The "no backups yet" shortcut used a local file check that can't apply to a remote repo, so it always looked empty. Remote repos are now listed directly.
- **`restic init` ran on every backup against an existing off-site repo**, logging a scary (but harmless) `config file already exists`. BombVault now probes the repo first and only initialises it when it's actually missing.
- **Stateless containers showed a phantom appdata folder.** A container with no appdata mount (and no `/mnt/user/appdata/<name>` folder) listed that non-existent folder as a selected backup folder, which then backed up nothing. The conventional appdata default is now only offered when the folder actually exists; otherwise the container is treated as config-only.
