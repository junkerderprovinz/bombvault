Adds native restic remote backends (S3, restic REST server, Backblaze B2, SFTP) as backup targets — without needing rclone. Thanks to **@manilx** for the request.

## Added
- **Remote targets without rclone.** Point a Backup Path straight at a restic remote repo: `s3:s3.amazonaws.com/bucket/path`, `rest:http://host:8000/repo`, `b2:bucket:path`, or `sftp:user@host:/repo`. (BombVault already understood these URIs; this release adds the credential handling to go with them.)
- **Cloud credentials** (Settings → Cloud credentials): encrypted storage for S3 (access key / secret / region) and the restic REST server (username / password). BombVault passes them to restic as the standard environment variables it expects. Secrets are write-only — they're never shown again after saving, and leaving a secret field blank keeps the stored one; clearing all fields removes the config. rclone remains available for everything else.

Image: `ghcr.io/junkerderprovinz/bombvault:2.2.0` (also `:2`, `:2.2`, `:latest`), multi-arch (amd64 + arm64).
