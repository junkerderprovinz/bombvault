Adds the third backup domain: **Unraid flash backup**. BombVault now backs up and restores Docker containers, KVM/libvirt VMs **and** the Unraid USB flash — the container is feature-complete across all three domains.

## Added — Unraid flash backup

- **Back up the whole flash (`/boot`)** via restic: Unraid OS, license, array config (`super.dat`), shares, network, SSH and plugin config — everything on the USB. Incremental + deduplicated like the other domains.
- **Safe restore.** A flash snapshot is **extracted to a folder** (`/mnt/user/bombvault/flash-restore` by default) that you then copy onto a fresh USB. The live, running `/boot` is **never overwritten** — flash restore can't leave the server unbootable.
- **Flash tab** in the sidebar (revealed when you enable Flash under Settings → Domains): one-click "Back up flash now", a snapshot list, and per-snapshot extract-restore that shows exactly where the recovered files landed.
- **Scheduled flash backups** run automatically when a Flash schedule is set (per-domain everyN due-gate, scoped to flash).
- **New mount:** the template now mounts the whole flash (`/boot` → `/host/boot`, replacing the narrower templates-only mount it subsumes). Re-import the template (or add the mount) to enable Flash backup.

## Notes

- Flash backup needs the `/boot` mount above; without it the app reports a clear "flash not mounted" message instead of failing obscurely.
- Restore writes to `<flash repo>-restore` next to your flash repo path — it is a recovery staging folder, not a live overwrite.
- Container and VM backup/restore are unchanged from v1.0.0.
