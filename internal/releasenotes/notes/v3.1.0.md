Schedules move into the **Plans** tab and a long-standing "Failed to fetch" papercut on single backups is gone.

## Fixed
- **Single "Back up now" no longer shows "Failed to fetch" while the backup actually runs.** Container, VM and Flash single backups are now asynchronous: the request returns immediately and the backup runs detached, so backing up the reverse-proxy container the WebUI is served through can no longer sever the result mid-run. The button watches the recorded run and reports the real outcome. (reported by @manilx, hit on the nginx-proxy-manager container)
- **The button reads the outcome of the exact run it started.** It now correlates the result by a new run id captured at fire time instead of comparing the browser clock to the server's timestamp, so a skewed client clock can no longer match the wrong run or leave the button spinning.
- **Bulk VM backup no longer mislabels VMs as failed.** It retries briefly while the previous VM's single-backup guard is still releasing instead of immediately reporting "a backup is already running".

## Added
- **Schedules now live in the Plans tab and are editable there.** Set a per-domain cadence for Containers, VMs and Flash, with an optional "use the Containers schedule for VMs and Flash too" sync. (the schedule settings moved out of Settings)
- **One-click "Include all in schedule" / "Exclude all"** for Containers and VMs, so you no longer have to flip each row by hand. BombVault's own container is never scheduled.
- The new Plans/schedule strings are translated into all 26 locales.
