First stable release of BombVault — a self-hosted, Unraid-native web app for **backup and full disaster recovery** of Docker containers and KVM/libvirt VMs, powered by restic. It ships as a single static Go binary serving a JSON API and an embedded React UI. One-click **container** and **VM** backup & restore are live; Flash backup is on the roadmap.

## Highlights

- **One container.** A single static Go binary (JSON API + embedded React/Vite SPA) — no external runtime, no database server. State lives in SQLite under `/config`.
- **Restore is the star.** Containers are recreated against the Docker API so they reappear in the Unraid Docker tab exactly as before — same image, env, ports, volumes and **security settings** (User, Cap add/drop, Privileged, SecurityOpt, ReadonlyRootfs, NetworkMode, Devices). VMs are re-defined in the VM Manager with their disks **and UEFI NVRAM** reattached — even after the VM was deleted.
- **VM backup over SSH — no libvirt mount.** BombVault runs `virsh` on the host over `qemu+ssh://`, so it can never interfere with the host VM Manager (mounting libvirt paths on Unraid is fragile and can break the VM Manager).
- **Optional encryption, optional auth.** Restic encryption can be toggled on/off; a password can be set to require login (off by default for trusted-LAN use, sessions HMAC-signed from `APP_KEY`).

## Container backup & restore

- Appdata directories + the full container definition are backed up via restic (incremental, deduplicated).
- One-click or scheduled backup; one-click restore of a chosen snapshot.
- **Pre-flight conflict check** — before anything is stopped or removed, restore verifies the container's static IP and published host ports are free and aborts with a clear message instead of leaving a half-finished restore.
- A live-container re-check guards the destructive stop/remove step; the container is always restarted, even on error.

## VM backup & restore (KVM / libvirt, over SSH)

- **Graceful** (clean shutdown → back up disks → restart) or **Live snapshot** (external snapshot + `blockcommit`, with an always-running safety guarantee — a commit failure never destroys the VM).
- Disk image(s) + domain XML + **UEFI NVRAM** are captured; restore re-defines the VM over SSH and reattaches everything, with an OVMF master-var fallback so UEFI VMs always boot.
- Public key + connection test live in **Settings → VM Backup over SSH**. Full setup & networking guide: `docs/vm-backup-ssh-setup.md`.

## Scheduling

- Per-domain schedules (daily / weekly / every-N-days), evaluated by an in-process scheduler.
- Container **and** VM scheduled backups run automatically (each VM/container opts in); the every-N due-gate is scoped per domain.

## Storage, encryption & security

- Backup repository paths are configured **in the app** (Settings → Backup paths) and default to `/mnt/user/bombvault/{container,vms,flash}`.
- Restic encryption toggle; key derived from `APP_KEY` (HMAC-SHA256, domain-separated). Definitions are sealed with AES-256-GCM for disaster-recovery discovery.
- Path-containment validation on every configurable path; resource-name validation on every API route; argv-separated shell-outs (no shell) for ssh / virsh / restic; restic arg-injection guard (`--` + hex snapshot-id); strict CSP + `nosniff` + `X-Frame-Options` on every response. HTTPS by default (self-signed), `HTTP_ONLY` fallback.

## Requirements & notes

- `APP_KEY` (64 hex chars, `openssl rand -hex 32`) is required — **keep it safe**, losing it makes encrypted backups unrecoverable.
- VM backup needs SSH enabled on the host and the shown public key authorized; set `LIBVIRT_HOST` / `LIBVIRT_SSH_PORT` to match your network. No libvirt mount is used.
- **Flash backup** (the Unraid USB config) is not yet implemented — it is clearly marked *coming soon* in the UI and is the next milestone.
- BombVault holds root-equivalent control of the host (Docker socket + root SSH for VMs). Run it on a trusted, non-exposed network only.
