Fixes a permissions bug that could lock you out of your own backup folder over a network share, and keeps the whole backup folder copyable off-box.

## Fixed

- **Your backup folder stays readable for an off-box copy.** BombVault's engine runs as root, and restic writes each local repo (and BombVault's disaster-recovery definitions) owner-only — directories `0700`, files `0600`. On a backup folder shared over SMB that denied a non-root network user access to the *whole* folder, breaking a program that copies the backups to a second drive. After every backup BombVault now relaxes the local repo tree to be group/other readable (dirs `0755`, files `0644`), touching only the entries that need it, so it stays cheap. The repos are encrypted, so the looser mode exposes nothing. A folder a previous version locked down is healed on the next backup.

## Changed

- **Disaster-recovery definitions now live inside each repo** — container definitions in `<repo>/def`, VM definitions in `<repo>/vm-def` — instead of as separate `bombvault-defs` / `bombvault-vm-defs` folders next to it. This keeps the backup root uncluttered and makes a copy of a repo folder self-contained: its recovery definitions travel with it. Definitions in the old location are migrated into the repo automatically on the next backup (and the old folder removed once empty), and a restore from a backup made before this change still finds them via a fallback, so nothing is lost.
