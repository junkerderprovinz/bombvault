Fixes a permissions bug that could lock you out of your own backup folder over a network share.

## Fixed

- **Your backup share stays readable for an off-box copy.** Next to each backup repo, BombVault mirrors an encrypted disaster-recovery definition per container/VM (in `bombvault-defs` / `bombvault-vm-defs`) so a fresh install can rediscover and restore everything without its old database. Because the engine runs as root, these were written root-only (directory `0700`, files `0600`) — which could deny a non-root SMB/network user access to the *whole* backup folder and break a program that copies the backups to a second drive. They are now written world-readable (directory `0755`, files `0644`), in line with the restic repo and the export folder beside them; the definitions are always encrypted with your `APP_KEY`, so the looser mode exposes nothing. A directory or file an older version locked down is healed to the readable mode on the next backup.
