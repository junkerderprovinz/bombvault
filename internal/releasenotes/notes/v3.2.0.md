## Added
- **Restore is clearer, and "restore to a folder" now uses a folder picker.** The per-snapshot restore actions are consolidated into a single **Restore…** panel with three clear choices: restore **in place** (recreate the container), restore **individual files**, or restore the whole snapshot **to a folder**. "To a folder" now lets you pick the destination from a folder tree instead of typing a path (which could land somewhere unexpected, like `/mnt/restore`), pre-filled with a new **Default restore folder** you can set in Settings. (#21, thanks @manilx)
- **The running version is shown at the bottom of the sidebar** (and in the startup log banner). (#22, thanks @manilx)

## Fixed
- **The version shows in the log again.** `:latest` builds were logging the branch name ("main") instead of a real version; they now carry a proper version. (#22)
- **Toggles and checkboxes follow your accent colour** instead of a fixed green.
