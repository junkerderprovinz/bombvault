Fixes the recurring VM live-backup "already exists" error and makes config-only (data-less) containers restorable after deletion. Thanks to **@manilx** for both reports.

## Fixed
- **VM live backup failing on the second run ("external snapshot file … already exists").** `blockcommit` merges a live snapshot back into the base disk and switches the VM onto it, but doesn't delete the now-orphaned overlay file — so every successful live backup left a `*.bombvault-tmp` file behind, and the next snapshot refused to start. BombVault now deletes those stray overlay files (before a backup, recovering a VM already stuck, and after each successful live backup). Live backups are repeatable again without manual cleanup.
- **Restore a config-only container after it's deleted.** A container with no data folders is backed up as "config only" (its definition is saved, but there's no restic snapshot). Once deleted it shows under **Not installed** — and now has a **Recreate from saved config** button that rebuilds it from the stored definition (image, env, ports, volumes). An installed config-only container shows a short explanation in its Backups panel instead of a bare "No backups found".
