Per-container exclude patterns: keep the important data, skip the caches.

## Added

- **Exclude patterns for container backups.** (#36, thanks @strese) Each container now has an **Exclude patterns** field (in its advanced settings) where you can list, one per line, subdirectories to skip inside a backed-up volume. The classic case: back up Plex's `/config` but drop the huge, regenerable `.../Plex Media Server/Cache`, `Media` and `Metadata` folders, shrinking the backup a lot while keeping the database, preferences and library settings.
  - **Type the paths you see inside the container** (e.g. `/config/Library/Application Support/Plex Media Server/Cache`). BombVault translates them to what restic actually stores, so they match exactly, no restic glob knowledge required. A bare name like `.git` still matches at any depth, and advanced host/glob patterns are passed through untouched.
  - **Live preview:** as you type, each line shows what it resolves to and warns when a line would exclude nothing (e.g. a folder whose volume isn't in the backup), so you can trust the exclude took effect.
  - Changing excludes affects future snapshots only; already-captured data stays until it ages out of retention. Excludes are per container; VM and config backups are unaffected (a VM is an opaque disk image, so a subdirectory exclude doesn't apply there).
