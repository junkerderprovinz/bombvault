A small quality-of-life addition for flash backups: BombVault can now drop a plain `.zip` of your flash backup into a folder of your choosing, automatically, after every flash backup.

## Added

- **Scheduled flash zip export** — turn it on and, after each successful flash backup, BombVault writes the snapshot out as a plain `.zip` to a folder you pick (Settings → the flash options). Point it at a Syncthing or rclone folder and your bootable-USB backup leaves the server automatically, so it's there even when the server itself won't boot. You choose the retention: a single `flash-latest.zip` that's overwritten each time (smallest footprint, one always-current file), or keep the newest N timestamped `flash-<date>.zip` files as a rolling history. The zip is written atomically (a temp file is renamed into place), so a sync client never picks up a half-written archive, and an export failure is logged without ever failing the backup itself.
