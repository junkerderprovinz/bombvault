## Added
- **Stop other containers during a backup.** Each container now has a "Stop other containers" section: list other container names (one per line, for example a database), and BombVault stops them while this container is backed up, then starts them again afterwards. They are always restarted, even if the backup fails, and the dependencies come back up before the container that needs them. It uses the Docker API directly, so no shell script or extra tooling is required. This covers the Backrest-style "shut down certain containers before the backup and restart them after" use case.

Image: `ghcr.io/junkerderprovinz/bombvault:1.8.0` (also `:1`, `:1.8`, `:latest`), multi-arch (amd64 + arm64).
