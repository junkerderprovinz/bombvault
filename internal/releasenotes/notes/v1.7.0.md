More fixes from Community-Apps feedback. Thanks for the detailed reports!

## Fixed
- **Containers with no appdata no longer fail to back up.** A stateless container (no mapped data) used to fail with "Fatal: all source directories do not exist". It is now backed up "definition-only": its template and config are captured so it can be recreated on restore, and the run is recorded as a success. Missing source paths are skipped instead of aborting the whole backup.
- **VM live backup no longer chokes on a CD-ROM.** A live snapshot tried to snapshot every disk, including the CD-ROM ("external snapshot file for disk hdc already exists and is not a block device"). It now snapshots only the writable disks and sets `snapshot=no` on CD-ROM and read-only disks.
- **The dashboard run history now shows which container/VM failed, and why.** Each run shows the target name instead of an opaque id, and a failed run shows its error message inline.

## Changed
- A definition-only backup can be restored: the container is recreated from its captured definition, no snapshot required.
