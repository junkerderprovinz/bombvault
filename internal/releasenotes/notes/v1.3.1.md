A focused fix release for the VM-backup setup experience and a privacy scrub.

## Changed
- **The VMs tab is now hidden until you enable the VMs domain.** Previously it was always shown even when VM backup wasn't set up.
- **Enabling VMs now requires a working SSH connection.** When you turn the VMs domain on, BombVault verifies the `qemu+ssh://` connection first and refuses (with a clear message) if the key isn't authorized yet — so the tab can never appear "active" without a working key.

## Added
- **Step-by-step key setup in Settings → VM Backup over SSH:** one-time instructions plus a **ready-to-paste authorize command** (your public key inlined; it authorizes the key on the Unraid host and persists across reboots) and a link to the full setup & networking guide. No more guessing how the key works.

## Fixed (privacy)
- The container template's **VM Backup: Host** default is now the placeholder `192.168.x.x` (you replace it with your Unraid LAN IP) instead of a concrete address, and example IPs/ports were scrubbed from the template description and the SSH setup docs.

> **Re-import the template** (or edit the container) to pick up the new `VM Backup: Host` placeholder.
