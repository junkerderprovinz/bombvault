A small UX clarification for the off-site DR check button.

## Changed

- **The "Run off-site DR check" button now reads "Re-run off-site DR check"** once a successful off-site DR restore is on record (shown by the green "proven off-site" badge). It is a permanent manual control — always available whenever an off-site repo is configured — so the relabel makes clear it's an optional re-check you can run whenever you like, not a pending task that should disappear after the first run. Translated across all 26 languages.
