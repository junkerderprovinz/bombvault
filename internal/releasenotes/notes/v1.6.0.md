Backup notifications, including Matrix.

## Added
- **Notifications.** Get told when a backup finishes. Settings has a new "Notifications" card with a policy (never / only on failure / on success and failure) and three optional channels, all stored encrypted at rest:
  - **Webhook** with a format selector: Generic JSON, Discord, Slack, Gotify, or ntfy.
  - **Matrix**: posts to a room via the client-server API (homeserver URL, access token, room ID; the token is sent in the Authorization header, never in the URL).
  - **Healthchecks.io**: pings the check URL on success and the `/fail` endpoint on failure.

  A "Send test" button checks the current form before you save. Notifications fire for container, VM and flash backups, both manual and scheduled, and never affect the backup itself (best-effort, time-bounded).

Image: `ghcr.io/junkerderprovinz/bombvault:1.6.0` (also `:1`, `:1.6`, `:latest`), multi-arch (amd64 + arm64).
