Bug-fix release from Community-Apps testing feedback — thanks!

## Fixed
- **A backup no longer changes your container's run state.** A container that was **stopped** before a backup is now backed up in place and **left stopped** — previously it was started afterwards. A running container is still stopped for a consistent backup and restarted. (VMs already behaved this way.)
- **Restore recreates a container in its backed-up state**, instead of always starting it. A container that was stopped when backed up comes back stopped; one that was running comes back running. Hooks are skipped for a stopped container (nothing to exec into).

## Changed
- The **HTTP_ONLY** template note now says: when enabled, open `http://[IP]:[PORT:3000]/` (the WebUI link points to HTTPS by default).
