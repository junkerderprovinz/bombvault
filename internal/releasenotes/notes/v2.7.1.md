A small diagnostic improvement.

## Added
- **The running version is now printed in the startup log.** The boot banner and the "BOMBVAULT IS READY" box now show the image version (e.g. `BOMBVAULT v2.7.1 IS READY`), so you can confirm at a glance which build a container is actually running — handy when checking whether an update has been pulled. The version is stamped into the binary at build time.

Image: `ghcr.io/junkerderprovinz/bombvault:2.7.1` (also `:2`, `:2.7`, `:latest`) and `docker.io/junkerderprovinz/bombvault:2.7.1`, multi-arch (amd64 + arm64).
