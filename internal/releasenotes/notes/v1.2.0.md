Adds the four most-requested power features — off-site repos, retention, file-level restore — plus a CI boot smoke-test, and fixes a settings bug that could disable the auth password.

## Added

### Off-site backups (rclone + SMB/NFS)
- A Backup Path can now be a **remote restic backend** — `rclone:<remote>:<bucket>/path` for cloud (Backblaze B2, S3, Google Drive, …). Paste your rclone config under **Settings → Off-site (rclone)**; it is stored **AES-256-GCM-encrypted** (APP_KEY) and written to a `0600` file only for restic→rclone to read.
- **SMB/CIFS & NFS** need no rclone: mount the network share on Unraid and point a Backup Path at it.
- Local and remote repos are auto-initialised on first use.

### Retention
- **Settings → Retention**: keep-last / keep-daily / keep-weekly / keep-monthly. After each successful backup, restic prunes older snapshots to the policy (per-item, via restic's host+paths grouping). All zero = keep everything (off, the default).

### File-level restore (containers)
- Expand a snapshot's **Files**, filter, and **restore an individual file or folder** back to its original location — recover one broken config file without restoring the whole appdata. The include path is validated to stay within the backup mount.

### CI boot smoke-test
- CI now **boots the built image and curls `/api/health`** before pushing `:latest` / the release tag, so a non-booting image can never reach the registry.

## Fixed
- **Saving settings no longer wipes the auth password.** The settings save rebuilt the row without the password hash (and would also have dropped the new rclone config); separately-managed secrets are now preserved across a settings save.

## Notes
- Off-site, retention and file-level restore are all configured in the app — no template change is required for this release.
- `rclone` ships in the image already; no new dependency.

Image: `ghcr.io/junkerderprovinz/bombvault:1.2.0` (also `:1`, `:1.2`, `:latest`), multi-arch (amd64 + arm64).
